
### Problema detectado:
El patrón captura correctamente "CKAE1AE3, S100, PAX8 y p40 heterogéneo" pero:

1. **El modificador "heterogéneo" se detecta bien** (línea 1472):
   ```python
   modifier_match = re.search(r'^(.+?)\s+(heterog[eé]neo|focal|difuso)$', part, re.IGNORECASE)
   ```

2. **PERO p40 se normaliza en minúsculas en el PDF** ("p40") y el mapeo solo tiene "P40" en mayúsculas.

3. **El normalize_biomarker_name() funciona correctamente** porque hace `.upper()` (línea 1500):
   ```python
   raw_clean = raw_name.strip().upper()
   ```

4. **El verdadero problema:** cuando el patrón falla, se captura basura de contexto.

### Comportamiento actual observado:
- Texto: "CKAE1AE3, S100, PAX8 y p40 heterogéneo"
- Split por comas y "y": ["CKAE1AE3", "S100", "PAX8", "p40 heterogéneo"]
- Para "p40 heterogéneo":
  - Se detecta modificador: biomarker_raw = "p40", modifier = "HETEROGÉNEO"
  - Se normaliza: normalize_biomarker_name("p40") → "P40"
  - Se asigna: results["P40"] = "POSITIVO HETEROGÉNEO"

**ENTONCES EL CÓDIGO ACTUAL DEBERÍA FUNCIONAR.**
